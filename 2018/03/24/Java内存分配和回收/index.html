Java内存分配和回收

Veröffentlicht am 2018-03-24 | in jvm

1 简介

JVM 内存布局

  1. 线程独享内存
  • PC：程序计数器，线程切换时需要保存当前执行状态，PC用于记录线程将要执行的下一条指令。
  • 虚拟机栈：JVM在执行函数的过程中，需要存储局部变量、参数、返回值等，用到虚拟机栈
  • 本地方法栈：JVM调用本地方法执行，需要存储局部变量、参数、返回值等，用到本地方法栈
  1. 线程共享内存
  • heap：堆内存通常是JVM最大的内存，用于存储对象实体
  • 方法区：用于存放代码指令、常量等

GC Root

  • 虚拟机栈中的引用指向的对象
  • 本地方法栈中的引用指向的对象
  • 类中静态变量指向的对象
  • 类中静态方法使用的对象
  • 类中常量引用（final修饰）指向的常量

GC Root其实就是heap内存之外的内存中的一些引用指向的对象。堆中引用指向的对象属于间接的，不属于GC Root。

JVM 内存回收算法

标记清除 Mark-Sweep

算法

  1. 先标记哪些内存需要回收
  2. 清除标记要删除的内存

优缺点

  1. 内存碎片

复制

算法

  1. 划分出一部分区域用于给新生成的对象分配空间
  2. 划分出的一部分区域被用完之后，复制到另外一份空闲区域

    优缺点

  3. 内存中有一部分无法使用
  4. 对象移动多

标记整理 Mark-Compact

算法

  1. 标记哪些内存需要回收
  2. 将不被回收的内存整理到一起

    优缺点

  3. 减少内存碎片
  4. 对象移动少
