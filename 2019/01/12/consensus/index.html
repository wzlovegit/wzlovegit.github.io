consensus 共识

Posted on 2019-01-12 | In base

一气呵成了解分布式介绍了入门分布式系统的一点基本理论。在现有的网络基础设施条件下，CAP 理论中的 P 是不可避免的，因此我们需要在 C 和 A 中做选择。Base 理论选择了 AP，强调基本可用、最终一致性，后面我们讨论工业界常见的达到最终一致性的方案。在有些常见的系统中，选择 CP，如果获取不到一致性的值，则会直接导致不可用，本文从 paxos 入手介绍分布式共识算法。

分布式共识模型

分布式共识在具备容错能力的分布式系统中是需要解决的一个基本问题，它指在多个分布式节点就某变量的值达成一致，而且一旦达成一致，则该值就是最终确认的值，不会由于某个节点的异常导致该值变化。
分布式共识模型是对工业界需要解决的常见问题的抽象，比如：选主、加锁等。想必大家见识过不止一种选主手段，比如采用 zookeeper 进行选主；在 redis 中通过 redis-sentinel 进行选主。

共识算法

如果我们不做数据内核相关的工作，目前对该类算法不感兴趣并不影响我们日常的工作，然而简单了解一下却可以帮助我们了解到类似 zookeeper 这些分布式协调服务的能力边界，不至于认为它可以顺利解决所有分布式问题。

形象化描述

你和其余4个朋友共 5 人，相约每周日要去球馆打篮球。市区内有 30 个球馆在周日提供服务。现在你们周日要去打球了，你们几个通过手机相互联系，怎么确定最终大家都能选中同一个球馆呢？
